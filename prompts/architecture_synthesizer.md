# Architecture Synthesizer Prompt

Use the processed patterns to propose original architecture for your system Enterprise OS 360.

## Rules
- Do NOT copy patent claims.
- Design ORIGINAL modules with entities, events, states, workflows, permissions, audit, and integrations.
- Each module must solve a real business problem.
- Each module must be independent but integrable.
- The architecture must be modular, event-driven, and rule-based.
- Use patterns as inspiration, not as a blueprint.

## Expected structure per module

1. **Purpose** — What business problem does it solve?
2. **Entities** — Main business objects
3. **State machine** — States and transitions
4. **Events** — What triggers state changes
5. **Workflows** — Main flows
6. **Rules** — Business rules
7. **Permissions** — Access control
8. **Integrations** — External systems
9. **Audit** — Decision trail
10. **Inspiration** — Which patterns informed the design (without copying claims)
